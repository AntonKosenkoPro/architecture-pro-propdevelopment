### Требования к внешним интеграциям

Поскольку партнерская система получает доступ к инфраструктуре жилых комплексов и ПДн, интеграция должна быть строго регламентирована.

#### 3.1. Требования к безопасности

- **Изоляция персональных данных:** PropDevelopment не передает партнеру ФИО и контакты жильцов в открытом виде. Обмен происходит по суррогатным идентификаторам `user_id`.
- **Согласие на биометрию (ФЗ-152):** В мобильном приложении PropDevelopment должен быть реализован явный сбор юридически значимого согласия на обработку биометрических данных перед активацией функции "Интеллектуальный домофон" по лицу.
- **Защита канала:** Весь трафик между бэкендом PropDevelopment и Платформой партнера должен шифроваться с использованием TLS.
- **Никакого прямого доступа с устройств:** Мобильное приложение жильца не должен обращаться к API партнера напрямую. Все команды маршрутизируются через отдельный сервис PropDevelopment (паттерн Backend-for-Frontend) для контроля доступа, аудита и логирования.
- **Аудит-логирование:** Бэкенд PropDevelopment должен логировать все события открытия дверей/шлагбаумов (кто, когда, как) для разбора инцидентов.

#### 3.2. Протоколы аутентификации и авторизации

- **Партнёрская интеграция (PropDevelopment и Партнер):**
  - Использовать взаимную аутентификацию **TLS** для транспортного уровня.
  - Для авторизации запросов использовать **OAuth 2.0**. Наша система получает токен у партнера для выполнения действий от имени системы.
- **Клиентская интеграция (Mobile App и PropDevelopment):**
  - Использовать текущий **OAuth 2.0** с JWT-токенами. При запросе видео с домофона или отправке команды на открытие, `tenant-core-app` проверяет JWT токен пользователя и его права на конкретную квартиру/ЖК.


#### 3.3. Организация взаимодействия систем

Взаимодействие должно быть разделено на два паттерна: асинхронный (управление правами) и синхронный (управление устройствами в реальном времени).

- **Асинхронный обмен:**
  - При покупке/аренде квартиры или добавлении члена семьи, `tenant-core-app` формирует событие (через брокер сообщений).
  - `Smart Home Adapter` преобразует это событие и вызывает API Партнера, передавая `user_id`, `building_id`.

- **Синхронный обмен:**
  - **Команды:** При нажатии кнопки "Открыть" в приложении, запрос идет в наш бэкенд -> проверяются права -> бэкенд синхронно вызывает REST API партнера
  - **Видеовызов (Домофон):** При звонке в домофон партнерская платформа инициирует push-уведомление через сервисы Apple/Google. Приложение поднимает сессию **SIP/WebRTC**, чтобы видеопоток шёл защищённо и с минимальной задержкой.
